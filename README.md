# 白沙宋墓 M1 点击解谜游戏

这是一个以白沙宋墓第一号墓（M1）为核心材料的沉浸式点击解谜游戏原型。玩家扮演考古实习生，通过观察墓葬图像、点击热区、收集线索、复查记录并形成章节结论，逐步理解 M1 的空间、图像、遗存和考古判断。

当前仓库处于整理期：项目已经可以本地运行，但文档、素材和协作规则仍在规范化。

## 当前范围

当前主线只围绕 M1：

- 墓外 / 环境
- 墓门
- 甬道
- 前室
- 过道
- 后室
- 终章汇总

暂不把 M3、独立小游戏合集、未经校核的剧情推断并入当前主线。

## 本地运行

需要先安装 Node.js。进入项目根目录后运行：

```powershell
node dev-server.cjs
```

默认地址：

```text
http://127.0.0.1:4173/
```

当前运行入口主要由以下文件组成：

| 文件 | 作用 |
|---|---|
| `index.html` | 页面入口 |
| `styles.css` | 界面样式 |
| `game.js` | 交互逻辑 |
| `data/scenes.js` | 当前正式场景、热区和线索数据 |
| `assets/` | 当前运行使用的图片素材 |
| `dev-server.cjs` | 本地静态服务器 |

## 分支原则

`main` 只放稳定、可运行、已验收的内容。团队成员应按任务创建分支，不按个人长期占用分支。

推荐命名：

```text
content/clue-table-v1-1
feature/npc-fixed-dialogue
feature/minigame-dig
asset/rear-room-images
docs/prd-cleanup
archive/old-docs-cleanup
```

所有合并到 `main` 的内容都应满足：

1. 属于当前 M1 PRD 范围。
2. 能追溯到正式线索表、素材表或校核文档。
3. 不混入旧口径、未校核推断或无关原型。
4. 本地运行不报错，关键图片路径存在。

更详细的整理和协作规则见：

- `docs/仓库整理方案.md`
- `docs/team/协作流程.md`
