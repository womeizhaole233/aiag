## 1. 坐标规范说明

| 属性 | 规范 |
|------|------|
| 原点 | 图片左上角 `(0.000, 0.000)` |
| 坐标系 | 归一化坐标，范围 `[0.000, 1.000]` |
| 精度 | 保留 3 位小数 |
| 元素层级 | MAIN（主元素）、SUB（次元素）、ENV（环境参照） |

**元素类型说明：**
- `MAIN`：线索核心交互点，首次点击触发
- `SUB`：二次/三次检查触发的细节元素
- `ENV`：用于空间定位的参照物

            var hotspotGO = new GameObject(element.element_id);

## 附录：人工验证指南

### 验证工具推荐
- **Godot引擎**：直接使用Image节点加载图片，在编辑器中拖拽定位
- **Unity引擎**：使用Sprite Renderer + Collider在Scene视图中标注
- **通用工具**：Photoshop/GIMP（显示像素坐标，换算为归一化坐标）

### 验证流程
1. 在引擎中加载对应图片
2. 根据元素ID和名称找到提示位置
3. 精确测量元素几何中心
4. 计算归一化坐标 = (像素x / 图片宽度, 像素y / 图片高度)
5. 保留3位小数填入本文档
6. 设置verified=true
7. 在对应图片文件中测试点击/悬停交互精度

### 精度要求
- **主元素（MAIN）**：误差 ±0.02（覆盖至少90%元素面积）
- **次元素（SUB）**：误差 ±0.01（小巧细节需更精确）
- **环境参照（ENV）**：误差 ±0.03（大致位置即可）
