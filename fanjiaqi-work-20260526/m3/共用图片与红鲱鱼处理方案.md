# M3 共用图片与红鲱鱼处理方案 v1.1

## 一、共用图片标注方案

### 1. L06 与 L09 共用图片
**共用文件**: `03-南壁/L06（M3 南壁彩画装饰带）.png`

**标注方式**:
- L06（彩画装饰带）：元素坐标 y≈0.4，关注中部三朵四瓣花纹
- L09（铺作与牡丹纹）：元素坐标 y≈0.25，关注上部铺作间牡丹纹

**JSON 中标记**:
```json
{
  "clue_id": "L09",
  "parent_clue_id": "L06",
  "view_type": "局部视图 - 铺作区域"
}
```

**引擎实现建议**:
- 加载同一张图片
- 通过元素 y 坐标区分交互区域
- 或使用不同图层/蒙版高亮不同区域

### 2. 坐标区分示例
```
图片垂直分布（从上到下）:
- y=0.0-0.3: 上部铺作与牡丹纹（L09）
- y=0.3-0.5: 中部彩画装饰带（L06）
- y=0.5-1.0: 下部（未使用区域）
```

## 二、红鲱鱼标注方案

### H01 · M3 甬道壁面墨点
**描述**: 甬道东壁下部，距地面约一尺处，五个墨点排列成弧形

**标注方案（二选一）**:
a) **在 L01 图上标注**：在甬道东壁壁画下部（y≈0.8 区域）
b) **独立交互区域**：在引擎中单独创建 H01 热区，不对应实际图片元素

**推荐**: 方案 b) 避免玩家误认为墨点是壁画的一部分

**JSON 示例**:
```json
{
  "element_id": "H01-01",
  "level": "HID",
  "name": "甬道东壁下部墨点",
  "center": [0.400, 0.800],
  "type": "红鲱鱼",
  "interactable": true,
  "notes": "点击触发误导性细节，与主线无关"
}
```

### H02 · M3 前室地面砖缝草屑
**描述**: 前室地面某砖缝中嵌着一小段干枯草茎

**标注方案（二选一）**:
a) **在地面相关图上标注**：可在 L03（矮足柜）或 L08（砖砌桌）的地面区域标注

**推荐**: 方案 a) 在背景图中适当位置标注，增加沉浸感

**坐标建议**: 选择图中地面区域（y≈0.9），位置不影响主要线索

## 三、引擎实现技巧

```gdscript
# L06/L09 共用图片处理
var image = load("res://03-南壁/L06.png")
var texture = ImageTexture.create_from_image(image)

# L06 热区

area_l06.position = Vector2(0.5, 0.4) * image_size

# L09 热区
var area_l09 = Area2D.new()
area_l09.position = Vector2(0.5, 0.25) * image_size

# 红鲱鱼点击处理
func _on_h01_clicked():
    if clue_system.has_clue("H01"):
        dialogue.show("这是五个墨点，看起来像是油灯火星溅落的痕迹...")
        # 标记为已查看，避免重复触发
        clue_system.mark_viewed("H01")
```

## 四、设计意图

### 为何共用图片？
- **来源限制**：考古现场照片有限，L06 和 L09 实际拍摄范围可能重叠
- **解谜逻辑**：L06 关注彩画装饰带，L09 关注上部铺作，观察焦点不同

### 为何设置红鲱鱼？
- **避免过度推断**：防止玩家把所有细节都当成关键线索
- **增加真实感**：考古现场确实存在无关痕迹
- **考验判断力**：培养玩家区分"重要证据"与"背景噪音"的能力
