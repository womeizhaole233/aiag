# 考古墓葬点击解谜游戏：玩家视角场景交互脚本

> 目标：本表只服务第一版可玩原型，不是资料索引。玩家看到的是墓室场景、物件名称和自然语言反馈；`L01`、`R01` 等编号只给开发使用，不进入游戏 UI。

## 设计原则

- 默认开局携带：记录夹、手电、标尺。它们是考古队员随身工具，不作为背景图里的可拾取物。
- 每个场景只保留 3-6 个可点击物件。点击对象必须是底图中真实可见的门、墙、壁画、器物、砖缝、人物、人骨或出土记录。
- 特写图不作为图片按钮出现，只由玩家点击主底图中的物件后进入“靠近观察”“查看背面”“用工具复查”。
- 首次点击偏现场感，二次点击偏考古判断，最终通过线索组合推进。
- 误导线索不直接告诉玩家“这是误导”，而是通过复查文本把它排除。

## 场景推进

```text
墓道入口
  -> 点击远处墓门阴影，靠近墓门

墓门
  -> 点击门额，发现门额位置与装饰逻辑异常
  -> 用手电查看门额背面，解锁“门额背面的彩画”特写
  -> 点击封门砖缝，获得一条可疑但可排除的颗粒线索
  -> 门额异常 + 封门砖未见盗扰，进入甬道

甬道
  -> 点击顶部彩画，发现叠胜纹样拼接错位
  -> 点击起拱线，用标尺复查弧线与图案对位
  -> 点击甬道深处，进入前室

前室
  -> 点击桌上砖砌注子，记录器物形制
  -> 点击高瓶与瓶座，形成器物年代矛盾
  -> 点击砖砌桌侧面，确认这些物件不是后放的实物，而是壁画/砖雕叙事的一部分
  -> 进入过道

过道
  -> 点击破子棂窗，发现窗与空间功能不完全吻合
  -> 点击流苏，得到一条误导性方向线索
  -> 点击纪年题记，发现笔迹与壁面层次异常
  -> 点击北侧入口，进入后室

后室
  -> 点击妇人启门雕像，发现手势异常
  -> 用手电复查门缝与手指，确认不是普通推门姿态
  -> 点击北壁下部砖床，打开出土记录视角
  -> 查看地券、人骨与铁钉，形成最终考古判断
```

## 误导线索处理

| 误导对象 | 玩家会怎么误解 | 如何通过玩法排除 | 内部编号 |

| 过道流苏方向 | 以为指向隐藏通道 | 点击流苏后再点击破子棂窗/题记，系统反馈其更符合帷帐装饰逻辑，不构成路线证据。 | H03 |
| 前室单个器物 | 以为某一件器物就是谜底 | 必须同时点击注子、高瓶、桌侧面，才形成“器物组合年代矛盾”；单物件只给观察。 | H05 可部分吸收 |
| 题记年代 | 以为题记单独决定墓葬年代 | 手电复查提示墨色与壁面层次异常，必须与地券、器物、图像共同判断。 | R01 |

下一步不要做“图片列表”，而是把本脚本拆成 5 类数据：

```js
const GAME_DATA = {
  startScene: "approach",
  defaultTools: ["记录夹", "手电", "标尺"],

  scenes: {
    approach: {

    }
  },

  hotspots: {
    gate_lintel: {
      scene: "tomb_gate",

    }
  },

    gate_lintel_first: {
```

字段重点：

- `objectName`：玩家理解的物件名，显示给玩家。
- `internalClueIds`：开发用编号，不显示。
- `viewMode`：`main_scene / closeup / back_side / record_view / tool_review`。
- `requiresTools`：只使用默认携带工具，不在图上凭空放道具。
- `grants`：获得线索、打开特写、推进场景或点亮组合关系。
- `isRedHerring`：只给开发判断，玩家只看到“被排除的自然解释”。
