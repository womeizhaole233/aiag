# 考古墓葬点击解谜游戏设计数据表

> 项目：白沙宋墓 M1 点击解谜游戏  
> 类型：Web 端 2D 静态场景点击探索  
> 玩家身份：考古队员  
> 设计依据：`D:\软件\github skill\game-development`、`D:\考古游戏\图片-底图版\M1`  
> 制作阶段：数据设计，不生成 HTML 游戏

## 1. 设计原则

| 原则 | 落地方式 |
|---|---|
| 数据驱动 | 图片、热点、线索、物品、解锁条件全部进入统一 `GAME_DATA`，HTML 只负责渲染 |
| 场景状态机 | 游戏按“环境图 → 墓道 → 甬道 → 前室 → 过道 → 后室 → 终局报告”推进 |
| 渐进式观察 | 热点分为 MAIN、SUB、HID、ENV；SUB/HID 由工具或前置线索解锁 |
| 线索图谱 | 使用 L01-L23、H01-H06、R01-R09 组织主线、误导和组合推理 |
| 物品解锁 | 手电、标尺、放大镜、侧光灯、拓片纸、结构图、方位标、记录夹分别控制二级观察 |
| Web 性能 | 后续实现时按场景懒加载图片，不一次性加载 M1 全部大图 |
