## 一、全局线索原则

| 原则 | 说明 |
|---|---|
| P0 | 没有它，章节小结论不成立 |
| P1 | 用来解释 P0 的空间、器物、图像或制度意义 |
| H | 只用于误导排除或降级，不进入正向证明链 |
| E | 用来建立位置、朝向、空间序列和观察背景 |
| R | 由复查和组合生成，是章节小结论或终章总判断 |
| 剧情边界 | 主线剧情可以借用悬疑感，但不能把单个异常写成盗墓、谋杀、身份真相、机关密室或超自然解释 |
| 小游戏边界 | 小游戏只能确认、定位、对照、归类或降级已有线索，不能凭空制造新事实 |

## 九、给主线剧情与小游戏组的使用提示

| 团队 | 使用方式 |
|---|---|
| 主线剧情 | 只把 P0/R 作为剧情推进依据；P1 用作解释和氛围；H 只能写成疑问和误判，不可写成真相。 |
| NPC 固定剧情 | NPC 可以提示复查动作、证据边界和缺项；不能替玩家直接得出章节结论。 |
| NPC 自由对话 | 回答应基于玩家已收集线索；如果玩家问超出证据的问题，NPC 应提示“还不能判断”。 |
| 小游戏 | 小游戏绑定复查动作：定位、辨读、归类、对照、降级；完成后产出记录或复查状态，而不是创造新证据。 |
| 场景/图片 | 本表暂不绑定热点；后续做场景接入时，再从本表反推图片、视角和点击对象。 |

## 十、下一步建议

1. 先由主线剧情和小游戏同学审阅本母表，确认每章“能写什么、不能写什么”。
2. 再逐章精修点击弹窗和记录夹文案，优先顺序建议：墓门 -> 甬道 -> 前室 -> 过道 -> 后室 -> 终章。
3. 等线索本体稳定后，再回到图片/热点映射和代码接入。
