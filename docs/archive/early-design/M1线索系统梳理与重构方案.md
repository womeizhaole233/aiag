# M1线索系统梳理与重构方案

## 一、工作结论

- 当前运行版已经从单纯场景浏览扩展到 `记录`、`结论卡`、`NPC`、`章节收束` 多套界面与流程，但底层仍然是“点击一次即收录一条记录”的线性模式，玩家几乎不需要做排除、复查、比对、归纳。
- 运行版 `scenes.js` 中实际可收录记录共 `82` 条，分布为：`墓门 6`、`甬道 6`、`过道 10`、`前室 24`、`后室 36`。原始导出见 `D:\考古游戏\analysis-m1-current-clues.tsv`。
- 现代码里已经埋有可承接重构的骨架：`结论卡`、`线索墙`、`章节完成`、`NPC 节点对话`、`记录夹分栏`。但它们目前主要是“点击后的展示层”，尚未变成“分析驱动层”。
- 项目外部资料 `D:\考古游戏\白沙宋墓-简体版` 已提供完整的专业支撑，不仅有单点知识技能，还有两份直接面向游戏的线索设计文档：
  - `game-clues-design.md`
  - `game-clues-deepened-v3.md`
- 最重要的发现不是“资料不够”，而是“现有运行版只用了资料里最浅的一层观察信息”，尚未接入：
  - 工具复查
  - 误导线索待验证
  - 跨图对照
  - 组合关系解锁
  - 假说提交与证据反驳

## 二、现有版本下线处理建议

### 目标

- 将当前版本明确标记为“点击式原型归档版”，停止继续把它当作可长期迭代的正式玩法底座。
- 保留现有素材、场景组织、结论卡文案和可复用界面代码资产，作为下一版“分析驱动原型”的资源库。

### 建议动作

1. 产品层下线
- 关闭当前版本的对外测试入口，或在入口页明确标识为“已归档原型，不再作为正式体验版本”。
- 暂停继续为当前版本追加普通点击热点，避免把错误核心循环继续做厚。

2. 数据层归档
- 保留 `data/scenes.js`、`game.js`、`data/game-data.json`、`考古墓葬点击解谜游戏设计数据表.md` 作为“原型基线”。
- 以当前版本为基点打一个归档标记，建议内部命名为 `prototype-click-archive`。

3. 存档层处理
- 当前存档基于 `localStorage`，建议在下一版切换新 `SAVE_KEY`，避免旧版存档污染新版分析状态。
- 如需继续本地测试，保留旧存档读取逻辑，但新版不复用旧结构。

4. 研发层切换
- 后续开发不再以“加更多热点”为目标，而改为“把现有 82 条记录组织成可复查、可排除、可组合的推理系统”。
- 现有 UI 可继续用，现有线索文案可继续用，但“记录生成逻辑”和“章节推进逻辑”需要重构。

## 三、当前运行版问题诊断

### 1. 核心循环过浅

- 当前循环基本是：点击热点 -> 立即收录 -> 自动推进。
- 玩家缺少“这条信息是否可靠”“它与哪条线索冲突”“是否需要工具复查”的主动判断。

### 2. 线索层级被压平

- 现有运行版把总览、近景、细部都直接当作并列记录。
- 但外部设计文档已经明确区分 `MAIN / SUB / HID / ENV / ACT` 五层互动，当前运行版没有真正落回这套结构。

### 3. 误导机制缺失

- 当前版本虽有若干“可能误导”的信息，如石英砂、流苏、尖鞋、灯菜，但运行时没有“待验证 / 已排除 / 误判惩罚”系统。
- 玩家因此不会形成考古式的“谨慎判断”，而会默认“点击到的都是有效主线”。

### 4. 结论卡偏展示，不偏推理

- 当前结论卡和线索墙已经能显示缺失证据，但仍然是“系统告诉你缺什么”。
- 玩家还没有经历“自己先提出假说，再被证据支持或否决”的过程。

### 5. 分析交互与考古判断尚未真正闭环

- 现有交互虽然开始尝试分类、排序、对照，但更多仍停留在展示层。
- 它们还没有深度绑定：
  - 工具复查
  - 误导排除
  - 跨图比对
  - 假说修正

## 六、建议的融合方案

### 6.1 数据层重构原则

- 不再把 `record` 当作最终线索，而把它改为“观察记录”。
- 在线索系统中新增四类对象：
  - `observation`：现场观察
  - `check`：工具复查
  - `hypothesis`：玩家假说
  - `relation`：组合推理链

### 6.2 现代码最适合如何承接

| 现有系统 | 当前作用 | 建议改造 |
|---|---|---|
| `记录` | 收纳点击文本 | 改成“观察记录夹”，分观察、复查、待验证、已排除四栏 |
| `结论卡` | 显示章节结论 | 改成“章节假说卡”，需玩家主动提交组合后才生成 |
| `分析交互` | 顺序/配对式展示 | 改成考古动作：测量、对照、标注、复原、排除 |
| `NPC 对话` | 入口/完成提示 | 改成“方法提醒”与“审稿式反馈”，不再直接给判断 |
| `messageLayer` | 点击即收录 | 改成“观察层”，允许标记为主线、误导、待复查 |

### 6.3 推荐的融合优先级

1. 先恢复 `data/game-data.json` 与 `考古墓葬点击解谜游戏设计数据表.md` 中的主线/误导/工具分层
2. 再用 `game-clues-deepened-v3.md` 给已有主线补二级细节与红鲱鱼
3. 最后接入更专业的结构、彩画、风水、制度解释型技能，避免一开始信息过饱和

## 七、多版线索关联策划方案

### 方案 A：观察 -> 复查 -> 组合 的标准考古版

#### 核心循环

- 点击热点，获得“观察记录”
- 选择是否标记为：
  - 主线候选
  - 待验证
  - 误导嫌疑
- 使用工具做二级复查
- 将复查后的记录拖入章节假说槽
- 假说满足条件后，生成结论卡

#### 触发联动逻辑

- `墓门门额` + `门额背面彩画` -> 解锁“门额前后不一致”
- `封门砖缝` + `门前地面` + 后续工具测量 -> 解锁“封门方式判断”
- `甬道顶部` + `叠胜彩画` + `甬道顶线` -> 解锁“顶部重整链”
- `注子` + `高瓶` + `桌侧线条` -> 解锁“前室器物年代张力”
- `题记文字` + `题记下沿` + `顶部宝盖` -> 解锁“过道时间层”
- `地券` + `人骨铁钉` + `分布图整体` -> 解锁“后室异常葬仪”

#### 交叉验证规则

- 只有总览信息，不允许直接生成结论
- 近景与细部至少各一条，才可进入组合
- 误导类信息必须经过一次工具复查或对照，才能被排除

#### 组合后整合信息

- 玩家不再得到“你已经完成章节”，而是得到：
  - 哪个假说被支持
  - 哪条信息仍冲突
  - 哪条误导已被降级

#### 最终剧情与考古认知成果

- 最终不是“点完全部热点”，而是得出：
  - M1 并非单一时点留下的静态墓葬信息
  - 装饰、文字、遗存和空间叙事可能经历不同阶段调整
  - 玩家完成的是一份阶段性考古判断，而不是侦探式唯一真相

#### 适用性判断

- 最适合作为下一版正式主方案
- 能最大程度复用当前 `结论卡`、`记录夹`、`NPC` 代码

### 方案 B：假说板驱动版

#### 核心循环

- 玩家每章先拿到 2 到 3 个“待验证假说”
- 例如前室：
  - 假说 1：前室是礼仪展示空间
  - 假说 2：前室存在年代错层
  - 假说 3：前室局部图像可能被后期调整
- 玩家必须把观察记录拖入假说板中
- 系统不告诉玩家标准答案，只提示“证据不足 / 证据冲突 / 误导未排除”

#### 触发联动逻辑

- 同一条记录可支持多个假说，但权重不同
- 例如 `高瓶与瓶座` 可同时影响：
  - 礼仪陈设
  - 年代错层
- `女乐尖鞋` 默认进入“待验证”，不能直接推动主线

#### 交叉验证规则

- 每个假说必须同时包含：
  - 1 条主图观察
  - 1 条细部观察
  - 1 条空间/结构信息
- 如果全是器物或全是图像，假说不能提交

#### 组合后整合信息

- 提交后不生成唯一结论，而生成：
  - “已支持”
  - “尚可争议”
  - “证据不足”

#### 最终剧情与考古认知成果

- 玩家学到的不是“谜题答案”，而是考古判断的等级差异：
  - 哪些结论能成立
  - 哪些只能暂时成立
  - 哪些必须继续保留争议

#### 适用性判断

- 学术气质最好
- 但第一次上手门槛高，建议作为方案 A 的第二阶段升级

### 方案 C：误导排除与审稿反馈版

#### 核心循环

- 玩家先收集大量观察
- 系统强制要求先处理“误导线索待验证”
- 只有误导被排除或降级后，真正的组合推理才会开放

#### 触发联动逻辑

- `石英砂`、`流苏`、`尖鞋`、`灯菜`、`白瓷碗` 先进入“待验证”
- 玩家需用：
  - 标尺
  - 放大镜
  - 结构图
  - 双图对照
 进行排除

#### 交叉验证规则

- 如果玩家把误导线索强行提交为主链证据：
  - 教授不给通过
  - 只提示“该判断过度依赖单点异常”
- 玩家必须补齐对照材料后，才可继续章节推进

#### 组合后整合信息

- 每章结束时不是“结论卡生成”，而是“教授审稿反馈”：
  - 这一章你判断得太快了
  - 哪条证据可以保留
  - 哪条证据该降权

#### 最终剧情与考古认知成果

- 强调考古方法中的“排除法”和“谨慎断言”
- 玩家会更清楚理解：
  - 不是所有异常都能导向真相
  - 过度解释本身也是错误

#### 适用性判断

- 最适合作为方案 A 的中后期叠加层
- 不建议单独做成唯一主方案，否则节奏偏慢

## 八、推荐路线

### 推荐采用：A 为主，C 为辅，B 做终章升级

#### 第一阶段
