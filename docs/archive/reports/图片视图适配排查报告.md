# 图片视图适配排查报告

## 排查结论

本次问题来自全局场景图片显示策略：图片与 SVG 热点层原本按安全区整块铺满，容易让贴近边缘的交互点在不同设备上贴边、过大或受界面控件挤压。排查范围覆盖 `data/scenes.js` 中全部实际游戏视图，共 46 个场景视图、215 个基础热点检查位（含每个视图固定的位置图入口）。

调整后采用统一场景画框：PC / 笔记本 / 平板 / 手机会按视口、安全区、图片原始比例重新计算前景图尺寸；全屏沉浸感由模糊背景继续承载，前景图片保留足够画面比例和操作留白。所有热点视觉坐标保持不变，点击命中区自动补足到 44px 触控安全尺寸，并限制在图片边界内。

## 适配规则

- PC 与大屏：前景画框最大约占安全区宽度 88%、高度 90%。
- 平板与窄屏：前景画框最大约占安全区宽度 90%、高度 88%。
- 手机：前景画框最大约占安全区宽度 92%、高度 90%，并避开顶部工具栏。
- 所有设备：热点坐标保持归一化映射，透明命中区最小 44px，命中区不越出图片边界。
