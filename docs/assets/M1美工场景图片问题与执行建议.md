## 5. 本地与分支保存说明

只要这个 Markdown 文件被创建在当前仓库目录中，本地就会马上拥有这个文件。

但需要区分三个状态：

1. 工作区文件：本地已经有文件，但还没有提交。
2. Git 提交：文件已经真正记录到当前分支历史中。
3. 推送到 GitHub：远程 `feature/scene-art-m1` 分支也能看到这个文件。

因此，保存到当前分支的完整流程应该是：

1. 创建或修改 Markdown 文件。
2. `git add` 暂存文件。
3. `git commit` 提交到当前分支。
4. `git push` 推送到 GitHub 的 `feature/scene-art-m1`。

如果只是创建文件但不提交，本地仍然能看到这个 Markdown，但它还不算正式进入分支历史，也不会出现在 GitHub 远程分支中。
