## 4. 当前统一口径

1. 总表和 ID 字典当前均为 76 个正式 clueId。
2. `END-P0-01/02/03` 是终章三条 P0 证据链，`END-R-01` 才是最终三栏报告。
3. `PASS-P0-03` 文案可称“丁字盝顶式宝盖”，但素材路径按现有文件名写“盡顶式”。
4. 独立 `NPC剧情与小游戏` 板块不在本包内，但总表保留 `NPC绑定` 和 `小游戏绑定` 字段用于线索触发对接。

## 5. 剧情与证据分栏口径

线索交付表中的“结论边界”“需分栏或暂不定案内容”，只表示这些内容不能直接进入“可判断”的学术结论栏，不等于禁止编剧写人物猜测、剧情悬念或存疑讨论。

当前真正硬性禁止只有三条：

1. 把题记一条线索写成解释整座墓的答案。
2. 把妇人启门、假门写成真实通道或密室入口。
3. 让 NPC 直接告诉玩家本章 R 结论。

其他内容，例如 F 线索、H 线索、人骨、铁钉、地券、身份、死因、迁葬、盗扰、官方史实等，可以作为人物猜测、剧情悬念、可推测或仍存疑处理。写入文档或剧本时需要标清它属于“剧情推测 / 仍存疑 / 待确认”，不要直接冒充为“可判断”的学术结论。

## 6. 一句话版本

策划看 `05_剧情体验交付` 和章节精修，编剧看双用线索和回收链，程序看总表和 `06_ID与校核交付`，美术看总表来源图和热点映射，学术校核看双源校核和 v1.1 校核报告。所有团队遇到冲突时，以总表、ID 字典和 v1.1 校核报告为准。
