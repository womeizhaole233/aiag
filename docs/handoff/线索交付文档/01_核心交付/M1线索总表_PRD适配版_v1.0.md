# M1 线索总表 PRD 适配版 v1.0

> 生成日期：2026-05-31
> 适配依据：`M1初步需求文档_PRD.md`、`M1各章节线索母表_剧情深化版_v1.1.md`、`M1全场景线索深化检查与结论收束_v1.1.md`、`M1/M1-全局线索关系矩阵-v1.2_谨慎收束版.md`。
> 使用原则：P0/P1/E/R 可进入学术证据链；H 进入结论前需要复查、降级或存疑；F 可进入剧情线、可推测栏或存疑栏，但不能直接覆盖可判断学术结论。

> 阅读补充：本表“结论边界”中的“不能推出 / 不能写成”，表示不能直接进入“可判断”的学术结论栏，不等于禁止剧情创作。除“题记单独解释整座墓”“妇人启门/假门写成真实通道或密室入口”“NPC 直接告诉玩家本章 R 结论”三条硬禁止外，其余高风险内容可以作为人物猜测、剧情悬念、可推测或仍存疑处理。

## 一、字段说明

| 字段 | 用法 |
|---|---|
| clueId | 统一线索编号，按章节前缀组织。 |
| 场景 | 所属章节或空间。 |
| 层级 | PRD 线索等级：P0/P1/P2/H/E/R/F/待核。 |
| 是否主线 | 是否影响当前单线流程推进。 |
| 来源图片 | 直接使用的图片；剧情线无图时标“剧情源/待补素材”。 |
| 观察文案 | 玩家点击后看到的事实性文案方向。 |
| 记录文案 | 写入记录夹的短文案方向。 |
| 复查动作 | 玩家在记录夹、小游戏或近景中要做的动作。 |
| NPC绑定 | 推荐触发或反馈 NPC。 |
| 小游戏绑定 | 对应 miniGameId；无则写“无”或“后续储备”。 |
| 结论边界 | 这条线索不能直接作为可判断学术结论推出什么。 |
| 进入哪条R链 | 进入的章节关系链或终章关系链。 |

## 五、当前适配结论

1. P0/P1/E/R 线索负责学术推进。
