> 参照依据：`02_章节精修交付/M1后室线索精修版_v1.0.md`、`03_后室专项交付/M1后室线索表_正式交付版_v1.0.md`、`03_后室专项交付/M1后室图片热点线索流程映射表_v1.0.md`
> 用途：给策划、编剧、程序、美术统一查询 `clueId`，避免旧编号、旧推断和旧线索口径回流。

## 0. 使用说明

1. 程序接入以本表 `clueId` 为稳定键，不再使用旧稿中的后室旧编号含义。
2. 策划设计解锁条件时，以 `P0/E/R` 为主线骨架，`P1` 作解释支撑，`H` 作复查降级，`F` 作剧情栏。
3. 编剧可以使用 `P0/P1` 触发剧情疑问，但不能改写其学术结论；`F` 可以承担人物情绪，但不能冒充考古证据。
4. 美术制作热点和近景时，优先核对“名称、场景、用途、不能推出什么”，避免把待核内容画成实证。
5. 本表的后室编号口径已与后室专项交付版对齐。

## 1. 层级边界速查

| 层级 | 含义 | 可以做什么 | 不能做什么 | 主要使用方 |
|---|---|---|---|---|
| P0 | 核心推进线索 | 支撑章节小结论、解锁关键复查、进入终章可判断栏 | 不能被剧情改写，不能单点推出人物动机或案件真相 | 策划、程序、编剧、学术校核 |
| P1 | 关键解释线索 | 解释 P0 的空间、图像、器物意义；增强章节理解 | 通常不单独卡关，不能替代 P0 生成结论 | 策划、美术、编剧 |
| P2 | 沉浸观察线索 | 增强现场感和可探索性 | 不能作为章节核心判断 | 美术、关卡策划 |
| H | 误导或待验证线索 | 制造疑问，要求复查、降级或排除 | 不能升级成正向证据，不能串成阴谋或灵异真相 | 关卡策划、编剧 |
| E | 环境线索 | 建立空间定位、入口、场景范围和观察前提 | 不能推出身份、死因或事件过程 | 策划、程序、美术 |
| R | 推理关系链 | 由多条 P0/P1/E 与 H 降级生成章节结论 | 不是普通点击热点，不应由单次点击直接获得 | 策划、程序 |
| F | 剧情虚构线索 | 推动人物、悬念、情绪和终章可推测栏 | 不能作为考古证据，不能覆盖 P0/P1 结论 | 编剧、剧情策划 |

## 6. 高风险旧说法分栏口径

以下内容不得直接进入“可判断”的正式线索正向结论，但可以按剧情需要进入人物猜测、可推测、仍存疑或待确认：

1. 四具人骨、八枚铁钉、十二行地券。
2. 题记后期添加、异常笔迹、伪造题记。
3. 面部破坏、身份抹除。
4. 地券涂盖、乡级行政区缺失异常、诅咒文本。
5. 后期进入、盗扰坐实、谋杀、风水诅咒、核心真相。
6. F 线索引出的墓主身份、死因、迁葬原因或官方史实。

当前硬性禁止只有三条：题记一条线索解释整座墓；妇人启门、假门写成真实通道或密室入口；NPC 直接告诉玩家本章 R 结论。

## 7. 交付口径结论

当前 M1 线索交付包的稳定口径是：

P0/P1/E/R 负责学术推进；H 负责误读复查和降级；F 负责剧情、人物和情绪。后室是证据最密集的场景，必须严格区分“安葬遗存与葬仪文书可判断”“人物经历和时间差可推测或存疑”“暗格、绢帛和私人自述为 F 剧情线”。团队后续新增内容时，应先查本 ID 字典，再写入总表或场景设计稿。
