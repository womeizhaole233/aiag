### 5.2 多图路径字段

部分来源图片字段使用中文分号 `；` 合并多个路径。给人阅读没问题，但程序读取时需要明确拆分规则。

建议程序对接前统一为以下两种之一：

| 方案 | 适用 |
|---|---|
| 继续 Markdown 表格，用 `<br>` 分隔多图 | 便于策划阅读，程序也较容易切分。 |
| 另建 JSON/CSV 素材映射表 | 便于程序直接接入，不依赖 Markdown 字符切分。 |

### 6.2 需提醒团队的点

规范与校核文档中仍会出现“身份抹除、风水诅咒、真实通道、后期添加”等词，但这些大多出现在：

1. 禁止清单；
2. 旧口径/新口径对照；
3. H 线索误读示例；
4. “不能写成什么”的边界栏。

建议在交付会上提醒团队：
看到这些词时，要先看上下文。它们不能直接进入“可判断”学术结论，但可以按剧情需要分栏为可推测、仍存疑或红鲱鱼降级对象。
