# 线索文案规范

本规范用于后续所有墓葬点击解谜关卡的线索文字撰写与修改。
适用范围包括弹窗文案、信息收集栏记录、近景观察文案。

## 一、核心原则

线索文字只描述玩家在图像中能够观察到的内容。

优先写：

- 图案
- 位置
- 形态
- 材质
- 色层
- 尺寸
- 层位
- 接缝
- 光影
- 保存状态
- 与相邻对象的可见关系

避免写：

- 开发说明
- 玩法说明
- 线索用途说明
- 研究过程说明
- 过度推理
- 直接替玩家生成本章 R 结论

## 二、必须依据材料

所有证据热区文案必须来自 `D:\考古游戏\游戏第一版\M1` 中的线索表、线索映射表或热点坐标映射表。

允许：

- 直接使用材料原文
- 将材料原文拆成短句
- 在不改变含义的前提下压缩材料文字
- 保留材料中的尺寸、材质、层位、形制、色彩和空间关系

不允许：

- 无依据新增事实
- 根据氛围自行编造推断
- 把普通观察写成主线证据
- 把材料没有说的内容写成确定事实

## 三、语言风格

文风应当是：

- 中文语境自然
- 简洁
- 有考古记录感
- 有少量悬疑感
- 不像翻译腔
- 不写空泛抒情

推荐句式：

```text
对象位于……
表面可见……
局部仍存……
边缘与……相接。
下方可见……
纹样以……勾勒。
色层与……不同。
```

不推荐句式：

```text
这说明……
这暗示……
这不像……
为什么……
背后似乎隐藏着……
玩家需要……
当前用于……
材料表将其列为……
该线索属于……
```

## 四、禁止出现的元叙述

弹窗和记录栏中尽量不要出现以下表达：

```text
当前未在材料表中承担独立主线线索
可作为……材料
当前用于……
用于辅助判断……
这一高度关系属于 L06 的二级细节
材料表将其列为……
该细节在材料中归入……
主线线索
支线线索
二级细节
sourceFile
sourceClueId
L05：
H01：
E04：
```

这些内容可以保留在数据字段中，但不应直接显示给玩家。

## 五、弹窗文案格式

弹窗文案建议控制在 2 到 3 句。

基本结构：

```text
第一句：说明对象和位置。
第二句：说明可见形态、材质、色彩或尺寸。
第三句：说明与相邻对象的可见关系。
```

示例：

```text
门额背面可见完整卷草纹彩画。
卷草以墨线勾勒，青绿填色，纹样从中心向两侧延伸，总长约一尺八寸。
背面朱红底色与正面白灰层质地不同，卷草纹下边缘有一道水平裁切线。
```

## 六、记录文案格式

信息收集栏记录应比弹窗更短，但仍保留关键事实。

推荐：

```text
门额背面有完整卷草纹彩画，总长约一尺八寸；背面朱红底色与正面白灰层质地不同，下缘见水平裁切线。
```

不推荐：

```text
L07：门额背面彩画提示墓门可能经过异常处理。
```

原因：

- 编号不应直接显示给玩家
- “提示异常处理”属于推理结论
- 记录栏应优先保存可见证据

## 七、悬疑感的处理

悬疑感应来自事实排列，而不是替玩家推理。

允许：

```text
正面为题字，背面为彩画，质地与图像均不相同。
```

不允许：

```text
如果墓门从未被调整，这幅图为什么藏在背后？
```

允许：

```text
题字周围留白较宽，人物与器物线条仍可辨认。
```

不允许：

```text
这不像装饰失误，更像一次沉默的改置。
```

## 八、推断性语言使用规则

尽量少用否定式排除。

少用：

```text
不能说明……
不足以证明……
并非……
不是……
```

如材料本身包含排除依据，可以保留事实，不要把排除过程写成玩家提示。

可改为：

```text
石英砂粒径约半分，与本地夯土中常见石英砂一致。
石英砂与石灰浆的胶结状态与周围灰缝一致。
```

而不是：

```text
这不是人工填塞标记。
```

## 九、近景文案规则

近景文案只描述被放大的局部。

推荐：

```text
瓶座下部刻三层莲瓣纹。
每层八瓣，瓣尖外卷。
莲瓣纹雕刻深度达三分。
```

不推荐：

```text
该观察需要与砖雕注子同尺度对比。
```

原因：这是玩法或分析指令，不是图像描述。

## 十、导航文案规则

导航热区可以有少量行动提示，但不要写成线索。

推荐：

```text
返回西壁后，案上器物与家具重新进入视线。
```

不推荐：

```text
需要回到西壁继续对照器物和家具。
```

原因：前者是玩家视角，后者是任务指令。

## 十一、source 字段规则

数据结构中应保留来源字段：

```js
sourceFile: "M1/05_前室_西壁/05_前室_西壁-增强线索表-v1.1.md",
sourceClueId: "L05"
```

但这些字段只供开发和核查使用，不应显示在弹窗或记录栏中。

## 十二、写作检查清单

每次新增或修改线索文案后，检查以下问题：

- 是否来自 M1 材料？
- 是否保留了至少一个硬事实？
- 是否只描述图像或可见现象？
- 是否删掉了“材料表、主线、二级细节、当前用于”等元叙述？
- 是否避免了无依据推断？
- 是否避免了翻译腔？
- 弹窗是否控制在 2 到 3 句？
- 记录栏是否比弹窗更短？
- sourceFile 和 sourceClueId 是否已写入数据字段？
