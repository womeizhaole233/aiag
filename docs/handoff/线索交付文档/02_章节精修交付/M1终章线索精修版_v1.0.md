# M1 终章线索精修版 v1.0

> 基于《M1各章节线索母表_v1.0.md》和已完成的墓门、甬道、前室、过道、后室精修版标准整理。
> 本版只打磨终章线索本身，暂不做图片、热点、坐标或代码映射。

## 一、章节定位

| 项目 | 内容 |
| --- | --- |
| 核心学术问题 | 终章如何把各章小结论组织成一个谨慎的大结论：M1 的意义是否来自单点发现，还是来自空间、图像、文书、遗物和遗骸的多类证据互相支撑？ |
| 终章大结论 | M1 不是靠某一条“关键异常”解释的墓葬。它应被理解为一个由空间序列、仿木构建筑意识、壁画功能分区、葬仪文书、实体遗存和日常器物图像共同构成的北宋砖室墓证据系统。 |
| 主线剧情可用方向 | 终章可以写成“提交阶段性考古报告”的高潮。教授引导玩家从“找答案”转向“说明证据如何支撑判断”，让剧情紧张感来自取舍和审稿，而不是突然揭露谋杀、盗墓、机关或超自然真相。 |
| 小游戏可用复查方向 | 终章线索墙、章节结论卡排序、证据类型归类、总证据链连线、H 线索降级墙、最终报告句子拼接。 |
| 证据边界 | 终章不新增前文没有支撑的事实，只汇总、连线、降级和表述。大结论不能超出各章 R 结论、END-P0/P1 证据链和 H 降级结果。 |

## 六、R 结论解锁条件

## 七、给主线剧情组的直接用法

| 节点 | 可写内容 | 不可写内容 |
| --- | --- | --- |
| 进入终章 | 队员把所有章节结论卡铺开，教授提醒“现在不是找新东西，而是看证据之间的关系”。 | 不新增神秘遗物、突然出现的密室提示或新人物供述。 |
| 空间链回望 | 写玩家从墓外一路回看至后室，意识到每一章都承担空间功能。 | 不把路线写成机关路线、逃生路线或追凶路线。 |
| 图像链讨论 | 写甬道、前室、过道、后室图像功能不同，体现空间分区。 | 不把所有壁画写成同一段剧情、同一场事件或墓主自传。 |
| 文书与遗存合读 | 写题记、地券、人骨、铁钉等证据类型不同，需要合读。 | 不让题记或人骨单独给出“全部真相”。 |
| H 线索审稿 | 教授要求删去或降级“太好看的猜测”，保留能被证据支撑的判断。 | 不把多个 H 线索串成阴谋、谋杀、盗墓或超自然解释。 |
| 终章收束 | 教授说明这是阶段性研究判断：M1 的价值在于多类材料共同指向。 | 不用“真相大白”“凶手揭晓”“机关解开”式结尾。 |

## 九、一页版收束语

终章的玩家任务不是找到一个“隐藏真相”，而是学会把整座 M1 的证据按权重组织起来。墓外和墓门建立进入序列，甬道提供过渡与引导，前室展示礼仪和人物陈设，过道提供时间锚点与空间转换，后室集中出现安葬核心、葬仪文书、遗存和寝居化图像。终章把这些章节小结论连成总证据网：空间序列链说明 M1 是完整系统，图像功能链说明不同空间承担不同表达任务，文书与遗存链提供实体基础，仿木构和器物链补充“地下宅第”与生活想象。所有 H 线索只能被降级、排除或保留为待验证，不能正向支撑大结论。最终，M1 的意义应被谨慎表述为：它是一座由空间、图像、文书、遗物和遗骸共同构成的北宋砖室墓证据系统，而不是一条单点异常可以解释的谜案。
