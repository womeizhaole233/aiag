# M1 线索交付文档清单

> 整理日期：2026-06-02
> 目录用途：同步给团队成员的线索交付包。原始文件未移动，本目录为复制整理版。
> 本次按要求取消 `NPC剧情与小游戏` 板块；NPC 固定剧情、自由对话和小游戏独立需求文档不放入本包。

## 一、阅读顺序

1. 先读根目录 `M1线索交付包_团队使用说明_v1.0.md`，确认不同团队各看哪些文件。
2. 再读 `01_核心交付/M1线索总表_PRD适配版_v1.0.md`，以此作为线索权威总入口。
3. 需要查 clueId、层级和变更记录时，读 `06_ID与校核交付/M1线索ID字典与变更记录_v1.0.md`。
4. 需要理解每章结论时，读 `01_核心交付/M1全场景线索深化检查与结论收束_v1.1.md`。
5. 需要做终章线索墙或 R 链时，读 `01_核心交付/M1-全局线索关系矩阵-v1.2_谨慎收束版.md`。
6. 落到每个场景执行时，读 `02_章节精修交付`。
7. 做后室时，必须额外读 `03_后室专项交付`。
8. 写剧情体验、场景节奏或过道/后室重点稿时，读 `05_剧情体验交付`。
9. 写文案、审剧情或校核学术边界时，读 `04_规范与校核`。
10. 需要看最新一致性校核结果时，读 `06_ID与校核交付/M1线索交付包二次一致性校核报告_v1.1.md`。

## 三、分类说明

| 分类 | 内容 | 主要使用方 |
|---|---|---|
| 根目录 | 交付清单、团队使用说明 | 所有人 |
| `01_核心交付` | PRD、总表、全场景结论、R链矩阵、章节母表 | 主策、程序、关卡、学术 |
| `02_章节精修交付` | 墓外、墓门、甬道、前室、过道、后室、终章章节稿 | 关卡、编剧、美术 |
| `03_后室专项交付` | 后室线索表、复查交互、图片热点映射 | 关卡、程序、美术、学术 |
| `04_规范与校核` | 学术/剧情校核、文案规范、可复用格式、AI提示词 | 编剧、学术、文案、AI协作 |
| `05_剧情体验交付` | 双用线索、体验节奏、主线回收链、过道与后室剧情化设计稿 | 编剧、关卡、UI/UX |
| `06_ID与校核交付` | ID字典、v1.0/v1.1一致性校核报告 | 程序、主策、学术、交付管理 |

## 四、交付边界

- P0/P1/E/R 可以进入学术证据链。
- H 必须复查、降级或排除，不能串成阴谋结论。
- F 只进入剧情线、可推测栏或存疑栏，不能覆盖考古结论。
- 不使用旧稿中的“四具人骨、八枚铁钉、十二行地券、涂盖、身份抹除、后期进入、风水诅咒、核心真相”等表述。
