# M1 文游融合实现说明

## 融合目标

本次调整不是把剧情元素追加到现有提示文案，而是把 `jvqing` 分支的文游属性接入当前 M1 主线：剧情以节点形式播放，保留说话人、长段叙事、选项、NPC 立绘，并在玩家搜索线索、完成小游戏、完成整理台章节判断后触发。

## 数据来源

- `baisha_chenyan/app.py`：原分支 `DIALOGUES` 共 92 个节点，已机械转换进 `data/jvqing-story.js`。
- `剧情文案/加入线索后的最终剧情_融合版.md`：补充当前主游戏按墓外、墓门、甬道、前室、过道、后室拆分后的 M1 场景节点。
- `剧情文案/人物立绘/`：接入林砚秋、粟柏年、周淼、苏池、陈怀远、赵老倔立绘。

## 当前接线

- `opening`：白沙开场、银扣、周淼/苏池接站、粟柏年初见。
- `scene_entry_*`：进入墓外、墓门、甬道、前室、过道、后室时播放入场文游节点。
- `clue_*`：关键线索触发 NPC 反应，例如墓门背面、甬道顶部、前室女乐、过道题记、后室假门/人骨。
- `puzzle_*_complete`：消消乐、甬道拼合、题记辨读、后室遗物定位后播放对应剧情。
- `scene_complete_*`：整理台完成分类、组合、降级并形成章节判断后，播放章节收束剧情，再允许下一关推进。
- `scene_complete_final_report`：终章三栏报告后播放封存与尾声节点。

## 保留边界

- 当前没有把 `baisha_chenyan` 的 Flask 页面作为第二套游戏入口，而是把其节点系统融合进主游戏浮层。
- 二号墓、三号墓等后续支线节点已保存在 `data/jvqing-story.js` 的原始节点中，但没有接入当前 M1 定位小游戏。
- 当前背后仍使用玩家所在场景底图；剧情节点里的 `backgroundImage` 被保留为数据字段，未替换主场景底图。
