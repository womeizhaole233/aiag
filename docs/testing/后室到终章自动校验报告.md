# 后室到终章自动校验报告

适用分支：`feature/scene-art-m1`

关联清单：`docs/testing/后室到终章手动试玩检查清单.md`

校验脚本：`scripts/validate-rear-final-flow.cjs`

运行方式：

```powershell
node scripts\validate-rear-final-flow.cjs
```

## 本轮结论

- 阻塞错误：0
- 非阻塞提醒：1
- 当前判断：后室到终章的代码条件没有发现会导致试玩卡住的硬错误。

## 已自动校验的内容

- 全局场景、视图、热区跳转目标是否存在。
- `transition.missingRecords` 中引用的记录、降级项、章节是否存在。
- 后室“汇总后室”门槛是否包含：
  - 假门图像复查。
  - 地券文书复查。
  - 葬具证据链复查。
  - 妇人手部断口降级。
  - 铁钉数量异常降级。
  - 后室组合判断。
- 后室葬具证据链是否仍为 7 条源证据，并且小面板步骤覆盖全部源证据。
- 后室组合判断是否仍为 5 条生成条件。
- 待验证降级是否等待正确复查：
  - 妇人手部断口等待假门图像复查。
  - 铁钉数量异常等待葬具证据链复查。
- 后室到终章的关键路线是否存在：
  - 后室入口总览 -> 后室北壁。
  - 后室北壁 -> 北壁下部。
  - 后室北壁 -> 人骨与铁钉。
  - 后室北壁 -> 出土物分布图 -> 人骨位置。
  - 后室北壁 -> 地券 -> 地券并盖。
- 终章入口与终章结论卡是否使用同一组章节完成条件。
- 终章入口是否必须等待后室组合判断。
- 终章入口在以下状态下是否正确锁定：
  - 空状态：应缺 6 项。
  - 只完成后室：仍应缺前四章。
  - 只完成前四章：仍应缺后室组合判断和汇总后室。
  - 五章完成且后室组合判断存在：应无缺失项。

## 非阻塞提醒

### `final-label-rear-only`

现象：终章入口的可见标签由 `navLabelCompletedSceneId: "rear_chamber"` 控制，因此在异常存档中，如果后室被标记为完成但前四章未完成，玩家可能看见“查看终章汇总”的标签。

影响：不构成提前解锁。点击入口时，`transition.missingRecords` 仍会检查墓门、甬道、前室、过道、后室五章完成状态和后室组合判断，未满足时会显示锁定提示。

处理建议：正常游玩流程下不会遇到。若后续支持跳关调试或导入旧存档，可以考虑把导航标签显示条件扩展为多条件判断。
