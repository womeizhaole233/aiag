# M1 线索总表模板

> 本表由“图片建模 + 线索”负责人维护，主策划/整合者审核当前单线流程中的必需线索。剧情组与小游戏组应引用本表中的 `clueId`，不得自行新增流程必需线索。

## 1. 填写原则

1. 真实考古线索必须能追溯到 M1 图片、线索表或映射表。
2. 必需线索必须支撑章节推进或章节结论。
3. 剧情虚构线索必须标为 F，不得混入真实考古证据链。
4. 每条线索必须说明是否影响推进。
5. 每条线索必须有弹窗文案和记录文案，文案遵守《线索文案规范.md》。

## 2. 线索分层

| 类型 | 当前流程作用 | 含义 |
|---|---|---|
| P0 | 必需 | 核心推进线索，缺少后无法完成章节判断 |
| P1 | 重要 | 关键解释线索，支撑章节结论 |
| P2 | 补充 | 沉浸观察线索，增加现场感 |
| H | 待验证 | 误导或需要降级的线索 |
| E | 环境 | 环境与空间定位线索 |
| R | 章节完成后生成 | 推理关系、章节结论或终章关系链 |
| F | 剧情 | 剧情虚构线索，只服务剧情互动 |

## 3. 线索总表字段

| 字段 | 填写说明 |
|---|---|
| clueId | 游戏内唯一编号，如 `gate_L01` |
| 章节 | 墓外 / 墓门 / 甬道 / 前室 / 过道 / 后室 / 终章 |
| 所属视角 | 正面 / 顶部 / 东壁 / 西壁 / 入口总览等 |
| 所属图片 imageId | 对应图片编号 |
| 线索名称 | 简短名称 |
| 线索类型 | P0 / P1 / P2 / H / E / R / F |
| 当前流程作用 | 必需 / 重要 / 补充 / 待验证 / 环境 / 剧情 |
| 是否影响推进 | 是 / 否 |
| sourceFile | 来源材料文件 |
| sourceClueId | 来源材料编号，没有则填“待补” |
| 事实描述 | 玩家点击后看到的事实，2-3 句 |
| 记录文案 | 进入记录夹的短句 |
| 关联线索 | 可组合的其他 clueId |
| 处理方式 | 收录 / 复查 / 降级 / 排除 / 汇总 |
| 可被 NPC 引用 | 是 / 否 |

## 5. 主线判断问题

每条线索进入主线前，需要回答：

```text
1. 它是否来自真实 M1 材料？
2. 它是否支撑本章核心判断？
3. 没有它，玩家是否仍能理解本章？
4. 它是否只是氛围、重复信息或单点异常？
5. 它是否属于剧情创作？
```

如果第 1 条为“否”，它不能进入真实考古主线。

如果第 2、3 条为“是”，它可以标为 P0 或 P1。

如果第 4 条为“是”，它更适合 P2 或 H。

如果第 5 条为“是”，它必须标为 F。
