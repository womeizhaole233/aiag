## 2. 当前理解

团队目前实际形成了三类工作：

1. 主项目 M1 的沉浸式点击解谜主线。
2. 可嵌入主线的小游戏原型。
3. 剧情向 NPC 互动脚本。

其中，M3 图片和线索整理已经有成员做过，但当前不继续扩展 M3。M3 只作为“后续候选素材储备”，不进入当前 M1 主线开发范围。

## 4. NPC 剧情互动需求

### 4.1 叙事定位

剧情互动用于增强沉浸感、解释考古工作流程、引导玩家理解章节目标。它不替代图片观察，也不直接替玩家生成本章 R 结论。

剧情应采用“考古现场叙事 + 玩家选择 + 反馈”的形式，接近橙光文游，但保持考古题材的克制感。

### 4.2 玩家身份

玩家扮演考古实习生或青年考古助理，第一次参与白沙宋墓 M1 的发掘与记录工作。

剧情稿中出现的“林同志 / 林助教”等称呼，可作为具体叙事版本的候选设定；主 PRD 中仍保留“考古实习生”作为通用身份。

### 4.3 NPC 角色抽取

| 剧情稿角色 | 可对应 PRD 角色 | 功能定位 |
|---|---|---|
| 粟柏年 | 考古领队 / 教授 | 章节目标说明、考古判断、选项反馈、结论收束 |
| 周淼 | 考古队员 | 壁画、纹饰、测绘、图像观察提示 |
| 陈怀远 | 建筑结构测绘人员 | 墓门、叠涩、砖砌结构、平剖面测绘提示 |
| 赵老倔 | 考古技工 / 当地技工 | 清理、拆砖、支撑、现场经验、地方口语氛围 |
| 苏池 | 体质人类学队员 | 人骨鉴定、年龄性别、迁葬相关判断 |
| 附近村民 | 墓外 NPC | 墓外传闻、发现经过、地方记忆 |

### 4.4 NPC 出场节点

| 阶段 | 出场 NPC | 作用 |
|---|---|---|
| 墓外 / 开场 | 村民、考古领队、考古队员 | 建立白沙地点、任务背景和考古队关系 |
| 墓道 / 墓门 | 领队、技工、结构测绘人员 | 引导观察夯层、封门砖、裂缝、拆除顺序 |
| 甬道 | 领队、测绘人员、壁画队员 | 引导观察版门、门钉、叠涩、顶部彩画 |
| 前室 | 领队、壁画队员 | 引导观察东壁女乐、西壁宴饮、顶部藻井 |
| 过道 | 领队 | 引导辨认纪年题记和破子棂窗 |
| 后室 | 领队、体质人类学队员、技工 | 引导观察妇人启门、地券、人骨、铁钉、遗物 |
| 章节收束 | 领队 | 组织玩家把线索转化为章节结论 |

### 4.5 剧情互动形式

每个剧情节点建议包含：

1. 场景描述。
2. NPC 提问或任务提示。
3. 玩家选择。
4. NPC 反馈。
5. 是否获得记录、解锁小游戏或推进场景。

示例结构：

```text
触发：玩家首次进入墓门区域。
NPC：考古领队、考古技工。
问题：封门砖应如何拆除？
玩家选择：先外层 / 先中层 / 先内层。
反馈：解释外层、中层、内层的支撑关系。
结果：获得“封门砖结构记录”，允许继续观察墓门。
```

## 5. 小游戏系统需求

### 5.1 总原则

小游戏必须嵌入墓葬调查流程，回答三个问题：

1. 它对应哪一个考古动作？
2. 它验证哪一条线索或哪一组线索？
3. 完成后推进什么场景、记录或结论？

### 5.2 已完成或已有原型的小游戏

| 小游戏 | 来源 | 当前状态 | 可进入 PRD 的定位 |
|---|---|---|---|
| 消消乐挖土 | 项目进度文档 | 已有核心玩法 | 可改造成墓外或墓门前清理土层小游戏 |
| 文物擦灰 | 项目进度文档 | 待设计 | 可作为文物清理、灰尘遮罩揭示类小游戏 |
| 墓门验证 | 范佳琦工作说明 | 已有探索原型 | 可作为墓门进入甬道前的线索验证 |
| 甬道叠胜纹校准 | 范佳琦工作说明 | 已有探索原型 | 可作为甬道进入前室前的图像校准 |
| 遗迹攀登 | 范佳琦工作说明 | 独立原型 | 暂列为远期支线，不进入当前 M1 主线 |

### 5.3 消消乐挖土需求抽取

该玩法可从“独立小游戏”调整为“墓外 / 墓门前清理动作”。

| 项目 | PRD 内容 |
|---|---|
| 触发场景 | 墓外、墓道、墓门前清理阶段 |
| 玩家操作 | 通过消除土块、碎石、残片、灰尘、砖块等完成清理 |
| 目标 | 清理足够区域，露出墓道、封门砖、地面痕迹或可点击底图 |
| 结果 | 解锁墓门场景或获得清理记录 |
| 失败处理 | 不做惩罚，允许继续消除或由 NPC 提示 |
| 是否影响推进 | 可作为墓外进入墓门的前置操作 |

### 5.4 文物擦灰需求抽取

| 项目 | PRD 内容 |
|---|---|
| 触发场景 | 文物出土、局部器物观察、记录夹复查 |
| 玩家操作 | 用刷子或擦拭动作清除灰尘遮罩 |
| 目标 | 逐步露出器物纹样、字迹或形制 |
| 结果 | 获得文物观察记录或近景证据图 |
| 可用场景 | 白瓷片、地券、题记、器物表面、墓门砖面 |
| 是否影响推进 | 第一版建议作为支线或复查动作，不强制卡主线 |

### 5.5 墓门验证小游戏需求抽取

| 项目 | PRD 内容 |
|---|---|
| 触发场景 | 墓门关键线索收集后 |
| 玩法形式 | 按正确顺序选择墓门相关线索 |
| 当前原型顺序 | 额 → 砂 → 雾 → 门 |
| 对应线索 | 门额背面彩画、封门砖缝石英砂、墓道墙面水汽、门洞深处 |
| 结果 | 验证墓门观察结果，开放甬道 |
| 注意 | “水汽”等内容需回到 M1 材料确认；若无材料支撑，不进入正式线索 |

### 5.6 甬道叠胜纹校准小游戏需求抽取

| 项目 | PRD 内容 |
|---|---|
| 触发场景 | 玩家观察甬道顶部叠胜彩画后 |
| 玩法形式 | 调整或校准叠胜纹样位置 |
| 对应线索 | 甬道顶部叠胜彩画、菱形单元、刮痕或重绘痕迹 |
| 结果 | 获得“叠胜纹校准记录”，可用于甬道章节收束 |
| 是否影响推进 | 可作为进入前室的增强条件，但需要避免过度卡关 |
| 注意 | 小游戏反馈应保持事实描述，不直接写成过度推断 |

### 5.7 远期小游戏方向

| 小游戏 | 可用场景 | 当前建议 |
|---|---|---|
| 器物归位 | 前室、后室 | 用于礼仪器物、日常器物归类 |
| 线索归档 | 记录夹 | 用于结构、图像、异常、红鲱鱼分类 |
| 拓片破译 | 墓门、过道、地券 | 用于文字、题记、铭文辨读 |
| 方位定位 | 墓外、全局定位图 | 用于墓道朝向、墓室结构判断 |
| 遗迹攀登 | 墓外或高处观察 | 远期支线，不进入当前 MVP |

M3 素材预处理 + 候选线索资料整理 + 后续三墓联动预留素材

2. 不把 M3 候选线索写入当前主线。

## 8. 团队协作需求

从成员工作说明中可以抽取出一个重要协作需求：必须建立统一 spec，否则图片、文案、代码、小游戏会各自推进。

### 8.1 统一规范内容

| 规范 | 作用 |
|---|---|
| 项目 PRD | 统一产品目标、范围和阶段 |
| 场景结构表 | 统一章节、视角、近景、返回关系 |
| 线索表 | 统一 clueId、sourceFile、sourceClueId、线索层级 |
| 图片资产清单 | 统一原始图片、已复制图片、已引用图片 |
| 小游戏设计表 | 统一触发条件、玩法、产出、是否推进 |
| NPC 剧情表 | 统一出场节点、对话目的、选择反馈 |
| Git 协作规则 | 统一提交范围，避免无关素材混入 |

### 8.2 小游戏接入规范

每个小游戏进入主项目之前，必须先写清：

```text
小游戏名称
对应章节
触发条件
玩家操作
成功条件
失败反馈
产出线索
是否影响推进
使用图片或素材
是否进入记录夹
```

## 10. 当前结论

团队成员的工作并不是与现有项目冲突，而是可以分成三类接入：

1. **立即可接入 PRD 的内容**：NPC 角色结构、小游戏接入原则、章节剧情节点。
2. **需要验证后接入的内容**：墓门验证中的具体词语、甬道校准中的图像解释、剧情稿中的部分考古细节。
3. **暂不接入的内容**：M3 继续开发、独立小游戏合集、bouldering-game 主线接入、虚构暗格线作为考古事实。

后续主项目仍应以 M1 为核心，把团队成果吸收为“剧情层”和“小游戏层”，而不是改变当前 M1 主线方向。
