# 剧情创作框架与 NPC 节点模板 v0.1

> 本文面向固定剧情 NPC 负责人、自由 NPC 负责人、主策划和整合人员。
> 本文不是完整剧本，不写死最终对白；它只规定剧情创作边界、章节必需节点和交付格式。
> 剧情同学可以在角色性格、对白风格、情绪节奏和局部表达上继续发挥，但不能越过 PRD、GDD 和线索交付文档的事实边界。

## 1. 依据文档

执行优先级：

```text
docs/design/GDD.md
docs/handoff/线索交付文档/01_核心交付/M1线索总表_PRD适配版_v1.0.md
docs/handoff/线索交付文档/01_核心交付/M1各章节线索母表_剧情深化版_v1.1.md
docs/handoff/线索交付文档/01_核心交付/M1-全局线索关系矩阵-v1.2_谨慎收束版.md
docs/handoff/线索交付文档/04_规范与校核/线索文案规范.md
```

遇到冲突时：

1. PRD 和 GDD 定当前版本范围。
2. 线索总表定 `clueId`、层级、边界和可进入的 R 链。
3. 剧情深化版定剧情悬念方向，但不覆盖学术事实。
4. 本文档定剧情同学如何交付。

## 2. 文档用途

本文解决四件事：

1. 告诉剧情同学哪些内容是硬规则，不能自由改。
2. 告诉剧情同学哪些地方可以发挥，允许后续迭代。
3. 按章节列出必须出现的 NPC 剧情节点位置。
4. 给出可直接填写的剧情节点表、角色表、F 线索表、H 降级表和 R 结论引导表。

本文不解决：

1. 不写完整主线剧本。
2. 不替剧情同学写所有对白。
3. 不把未经确认的历史事实、墓主身份、死因、迁葬原因或官方史实冒充为已确认结论；如需使用，必须标注为可推测、仍存疑或待确认。
4. 不决定最终 UI、配音、立绘和演出规格。

## 3. 创作分层

| 层级 | 是否固定 | 内容 | 谁负责 |
|---|---|---|---|
| 硬规则层 | 固定 | 章节顺序、线索层级、`clueId`、F/H/R 分栏规则、小游戏卡点 | 主策划 / GDD / 线索交付 |
| 节点结构层 | 半固定 | 每章何时出现 NPC、何时提示、何时收束、何时触发小游戏 | 主策划 + 剧情负责人 |
| 创作表达层 | 可迭代 | NPC 性格、对白节奏、玩家选项、情绪铺垫、人物关系细节 | 剧情负责人 |
| 演出实现层 | 待后续 | 立绘、镜头、音效、UI 表现、动效、配音 | 美术 / 程序 / 整合 |

## 4. 硬规则

剧情创作必须遵守：

1. NPC 不直接告诉玩家本章 R 结论。
2. NPC 可以提醒观察、解释方法、质疑过度推断、引导复查。
3. 真实考古事实只能来自线索交付表、PRD 或已确认材料。
4. P0/P1/E/R 可以进入学术证据链。
5. H 线索可以制造好奇、误读和讨论；进入结论前必须复查、降级或标为存疑。
6. F 线索可以承担剧情悬念、人物情绪和可推测内容；进入结论时必须与学术证据分栏。
7. 章节推进由线索、小游戏、记录夹和 R 结论卡共同完成，不能只靠 NPC 口头宣布。
8. 自由 NPC 只能在本章 `allowedClues` 范围内回答，不能提前剧透后续章节。

当前真正禁止写法只有：

1. 把题记一条线索写成解释整座墓的答案。
2. 把妇人启门、假门写成真实通道或密室入口。
3. 让 NPC 直接告诉玩家本章 R 结论。

其他高风险内容，例如 F 线索、H 线索、人骨、铁钉、地券、身份、死因、迁葬、盗扰、官方史实，可以作为人物猜测、剧情悬念、可推测或仍存疑处理。剧情负责人不需要把这些内容全部删掉，但必须让玩家知道它们还不是“可判断”的学术结论。

## 5. 可自由发挥范围

剧情同学可以自由发挥：

1. 角色说话方式、性格、口头习惯。
2. 玩家选项的措辞和情绪差异。
3. NPC 之间的轻微关系张力，例如严谨、好奇、谨慎、幽默、质疑。
4. 林砚秋的内心记录、困惑、成长感。
5. 每章开场和收束的情绪节奏。
6. F 线索的悬念呈现方式，但必须标注为剧情线、可推测或存疑。
7. 失败反馈、提示层级和重复提示的语气。
8. 后续版本中增加支线对白，但不能改变当前主线推进条件。

自由发挥时必须标注：

| 标注 | 含义 |
|---|---|
| 已确认事实 | 可进入学术链，必须能追溯 `clueId` |
| 方法提示 | NPC 引导玩家观察、复查、分类、降级 |
| 剧情推测 | 可服务情绪和人物线，但不能当事实 |
| 存疑 | 玩家可以记录疑问，但不能得出定论 |
| 待确认 | 需要主策划或线索负责人确认后才能进入正式版本 |

## 6. 固定 NPC 角色表

以下是当前第一版固定 NPC 角色方向。角色小传、口吻、关系和完整对白由剧情负责人继续补充。

| 角色名 | 类型 | 主要章节 | 必须承担的功能 | 可发挥空间 |
|---|---|---|---|---|
| 林砚秋 | 玩家角色 / 实习生 | 全流程 | 观察、记录、提问、承接个人剧情线 | 个人动机、内心独白、成长弧线 |
| 赵老倔 | 附近村民 | 墓外 / 环境 | 提供地方记忆、传闻氛围，引出传闻需降级 | 方言程度、说话节奏、传闻细节 |
| 陈怀远 | 结构 / 现场技术向 NPC | 墓门、甬道、过道、后室 | 解释封门、砖缝、构件、结构安全和真实/非真实通道边界 | 技术口吻、现场经验、与玩家互动 |
| 周淼 | 图像 / 壁画 / 文字观察向 NPC | 甬道、前室、过道、后室 | 引导壁画、器物、题记、反切、图文互读 | 观察敏感度、专业兴趣、对细节的兴奋感 |
| 苏池 | 遗存 / 人骨 / 器物分布向 NPC | 后室、终章 | 提醒人骨、铁钉、地券、随葬器物的解释边界 | 冷静程度、专业边界、对玩家猜测的纠正方式 |
| 粟柏年 | 考古领队 / 教授 | 章节收束、自由 NPC、终章 | 审核玩家判断、引导 R 卡、限制过度推断 | 语气、师生关系、最终审稿风格 |

## 7. 全局 NPC 节点类型

剧情节点不等于长对白。每个节点必须有清楚的游戏目的。

| 节点类型 | 出现时机 | 作用 | 是否必须绑定线索 |
|---|---|---|---|
| 章节入口节点 | 玩家首次进入章节 | 说明本章观察目标和现场状态 | 建议绑定 E 线索 |
| 关键线索节点 | 玩家点击 P0/P1 后 | 提醒复查动作或解释观察方法 | 必须绑定 `clueId` |
| H 降级节点 | 玩家误读或收集 H 后 | 引导玩家复查、降级、存疑 | 必须绑定 H 线索 |
| F 剧情节点 | 玩家触发 F 后 | 制造情绪或悬念，但标为剧情/存疑 | 必须绑定 F 线索 |
| 小游戏前节点 | 小游戏开始前 | 说明要做的考古动作 | 必须绑定 `miniGameId` |
| 小游戏后节点 | 小游戏完成后 | 反馈产出记录，避免把小游戏变成纯娱乐 | 必须绑定 `miniGameId` 和产出线索 |
| R 引导节点 | 玩家整理线索前 | 用问题引导玩家形成结论 | 必须绑定 R 卡 |
| 章节收束节点 | 玩家生成 R 卡后 | 承认玩家判断，解锁下一章 | 必须绑定 R 卡 |
| 缺项提示节点 | 玩家无法推进时 | 告诉玩家缺少哪类动作 | 绑定缺失线索或缺失动作 |

## 9. F 线索使用规则

F 线索可以写得有情绪，但必须保持“剧情可推测 / 仍存疑”的身份。

| F 线索 | 剧情作用 | 可以写 | 需要标注 / 不能直接定案 |
|---|---|---|---|
| `ENV-F-01` 外祖父银扣 | 林砚秋个人动机 | 私人记忆、进入调查的情绪原因、个人记录 | 若关联 M1 身份、年代或墓内事件，需标为个人推测或待确认 |
| `GATE-F-01` “永安”刻字 | 第一处文字悬念 | 疑似刻字、拓片、存疑、等待后续回看 | 若写堂号、口令、机关或身份猜测，需标为剧情推测 |
| `FRONT-F-01` 屏风隐字 | 从看图转向找文 | 疑似文字、侧光观察、候选字待核 | 若写完整历史事件或真实留言，需要标为待确认或后续材料 |
| `PASS-F-01` “赵大翁是谁” | 身份悬念 | 由题记称谓引发追问 | 不能只靠题记解释整座墓；身份内容需标为悬念或推测 |
| `HS-F-01` 假门凹槽与暗格 | 隐藏文书入口 | 剧情层隐藏物、复查后触发 | 不能把妇人启门、假门写成真实通道或密室入口 |
| `HS-F-02` 反切素帛 | 姓名拼合玩法 | 文字学方法、候选姓名、玩家参与 | 若写姓名结论，需标为剧情推测或待确认 |
| `HS-F-03` 赵怀诚绢帛 | 剧情自述 | 自述文本、残字、存疑、情绪高潮 | 若涉及官方史实、死因或墓主身份，需进入可推测或仍存疑 |
| `HS-F-04` 1099 / 1106 时间差 | 时间线疑问 | 阶段性安葬疑问、可推测栏 | 若涉及迁葬原因、初葬对象或案件，需进入仍存疑 |
| `END-F-01` 赵怀诚线收束 | 终章剧情余味 | 可推测、仍存疑、个人判断 | 与学术可判断栏分开呈现 |

F 线索写法公式：

```text
发现一个可疑点
→ NPC 或玩家承认它能引发疑问
→ 明确它目前不是学术证据
→ 放入剧情线 / 可推测 / 仍存疑
→ 等后续章节或终章再回看
```

## 10. H 线索降级剧情写法

H 线索不是“删除线索”，也不是禁止剧情使用。剧情同学可以写玩家产生好奇、争论和误读，但要写出 NPC 如何引导复查，以及复查后这条线索是降级、保留存疑，还是等待后续材料。

H 降级写法公式：

```text
玩家注意到异常
→ 玩家提出一个可能过强的猜测
→ NPC 要求换一种观察方法
→ 玩家完成对照 / 分类 / 叠合 / 放大
→ NPC 反馈：这个点可以记录，但不能承担强结论
→ 记录夹把它标为已降级或存疑
```

| H 类型 | 推荐 NPC 语气 | 处理边界 |
|---|---|---|
| 传闻类 | 赵老倔可讲得生动，粟柏年负责分栏 | 传闻可以保留氛围，但需标为口述、待验证或存疑 |
| 结构细节类 | 陈怀远要求回到整体结构 | 暗号、机关、盗扰等可作为猜测出现，但要经过复查或存疑 |
| 图像细节类 | 周淼要求放回整幅图像和同类细节 | 身份、民族、暗门等可作为误读方向，但不能越过三条硬禁止 |
| 文书过度解释类 | 粟柏年提醒文字是锚点，不是全部答案 | 不能只靠题记解释整座墓 |
| 遗存分布类 | 苏池要求层位、范围、叠合 | 死因、案件、二次开棺等可作为疑问，但需标为推测或存疑 |

H 降级节点交付时必须写明：

1. 玩家误读方向。
2. NPC 如何阻止过度解释。
3. 玩家需要做的复查动作。
4. 降级后的记录夹状态。
5. 这条 H 线索不能直接定案的内容。

## 11. R 结论卡引导写法

R 结论卡由玩家整理线索并提交相似判断后生成。剧情同学要写“引导问题”和“反馈”，不直接写答案宣布。

R 引导写法公式：

```text
NPC 提醒玩家回看几组证据
→ 玩家把线索卡放入指定证据组
→ NPC 提出问题
→ 玩家提交判断
→ 判断相似则生成标准 R 卡
→ NPC 认可方法，而不是替玩家宣布答案
```

| R 卡 | 引导问题方向 | 需要引导玩家处理的边界 |
|---|---|---|
| `ENV-R-01` | 这次调查从哪些真实地理和空间信息开始？ | 这座墓的身份或墓主是谁 |
| `GATE-R-01` | 墓门为什么既是入口也是证据界面？ | 墓门藏着机关或异常事件 |
| `COR-R-01` | 顶部、东壁、西壁合起来说明甬道承担什么功能？ | 某个单点异常证明后续扰动 |
| `FRONT-R-01` | 前室各墙面和顶部如何共同构成礼仪展示？ | 单一宴饮场景或唯一身份揭示 |
| `PASS-R-01` | 题记、窗格、顶部和图像导向共同说明过道是什么？ | 题记解释整座墓 |
| `HS-R-01` | 后室的遗存、文书、图像和器物如何叠合？ | 谋杀、盗墓、密室现场 |
| `END-R-01` | 哪些可判断，哪些可推测，哪些仍存疑？ | 唯一破案式真相 |

R 反馈分三类：

| 反馈类型 | 用途 | 写法 |
|---|---|---|
| 通过反馈 | 玩家判断接近目标 R | 肯定玩家使用了正确证据组合 |
| 缺项反馈 | 玩家少收或少复查 | 提醒缺少哪一类证据或动作 |
| 越界反馈 | 玩家把 F/H 写成事实 | 要求回到证据边界，放入可推测或存疑 |

## 18. 审核清单

主策划审核剧情稿时，优先检查以下问题：

1. 是否每个剧情节点都有 `nodeId`。
2. 是否每个关键对白都绑定章节和触发条件。
3. 是否所有事实性内容都有 `clueId` 或已确认来源。
4. 是否把 F 线索需要存疑的内容标为了可推测、仍存疑或待确认。
5. 是否给 H 线索安排了复查、降级或存疑处理。
6. 是否让 NPC 直接告诉玩家本章 R 结论。
7. 是否提前剧透后续章节。
8. 是否明确玩家在该节点之后获得什么记录或解锁什么内容。
9. 是否写明待确认问题，而不是强行定案。
10. 是否保留了后续迭代空间。

## 19. 推荐迭代方式

第一轮只交“节点粗稿”：

1. 节点位置。
2. 出场 NPC。
3. 绑定线索。
4. 节点目的。
5. 简短对白方向。
6. 待确认问题。

第二轮再补“对白细稿”：

1. 具体台词。
2. 玩家选项。
3. 正向反馈。
4. 纠偏反馈。
5. 情绪节奏。

第三轮再做“接入稿”：

1. `nodeId` 固定。
2. 触发条件固定。
3. 产出结果固定。
4. 与小游戏、记录夹、R 卡连接。

这样可以让剧情同学先发挥，再逐步收束到可接入版本。
